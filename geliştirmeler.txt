4-)Profil E-posta (/dashboard/profile/edit/)  << parola unutunca yenilemek için "http://127.0.0.1:8000/dashboard/login/" bölümü için
Kullanıcı talep eder: “Parolamı unuttum” diyerek e-posta girer.
Token oluşturulur: Rastgele, tek kullanımlık ve süreli bir kod üretilir.
E-posta gönderilir: Token içeren güvenli link kullanıcıya yollanır.
Token doğrulanır: Linke tıklayınca sistem tokeni ve süresini kontrol eder.
Yeni parola girilir: Kullanıcı yeni parolayı belirler, sistem hashleyip kaydeder.
Token geçersiz kılınır: Tekrar kullanılamaz.
Güvenlik için HTTPS, token süresi, tek kullanımlık token ve rate limiting şart.






şuan bunlar kaldır zorluk derecesine göre sırala benim için





1️⃣ Görselleri kırpma (Dashboard yükleme sonrası)

Zorluk: 🟡 Orta

Admin panelde kırpma alanı eklenip, yükleme sonrası Pillow veya Cloudinary transform ile farklı boyutlara göre kırpma yapılacak.

Her bölüm (galeri, ürün, carousel) için ayrı preset tanımlanacak.

Teknik ama Django’nun ImageField + django-imagekit veya Cloudinary API ile çözülebilir.

2️⃣ Depolama (Cloudinary Storage Progress Bar)

Zorluk: 🟡🟠 Orta – Zor

Cloudinary API’den toplam kullanılan depolama alanı çekilecek.

Resim + videoların boyutları toplanıp progress bar’a aktarılacak.

Hem backend (API çağrısı) hem frontend (bar güncelleme) yapılmalı.

Mantık basit ama entegrasyon testleri uğraştırabilir.

 

3️⃣ İstatistikler (Otomatik güncellemeli)

Zorluk: 🔴🔴 Zor

Index ve Hakkımızda bölümlerinde rakamların görünmemesi → template + context fix.

Dinamik istatistik:

Yorum geldikçe → tamamlanan iş artacak.

2+ puanlı yorumlar → mutlu müşteri.

Ortalama puan → memnuniyet.

Her yıl otomatik güncelleme.

Her giriş → tamamlanan iş artsın.

Backend mantığı + cron veya sinyal tabanlı güncelleme gerek.

4️⃣ Profil E-posta / Şifre sıfırlama (Token sistemi)

Zorluk: 🔴🔴🔴 En Zor

“Şifremi unuttum” akışı:

Kullanıcı e-posta girer.

Tek kullanımlık token üretilir.

E-posta gönderilir.

Linke tıklanınca token doğrulanır.

Yeni şifre girilir, hashlenir.

Token geçersiz kılınır.

Güvenlik: HTTPS, rate limit, expiry time.

Bu, güvenlik ve backend altyapısı gerektirdiği için en kritik iş.
